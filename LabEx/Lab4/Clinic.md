This state machine diagram represents the lifecycle of an appointment within a clinic management system, beginning in an "Idle" state until a patient requests a slot. It features a composite "Appointment Booking" state where availability checking and time confirmation happen sequentially before transitioning to a "Scheduled" status. From there, the appointment can either be "Cancelled" by a user or move seamlessly into the "In Progress" and "Completed" states once the actual consultation takes place.
